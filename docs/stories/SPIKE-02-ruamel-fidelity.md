# Spike: [SPIKE-02] ruamel.yaml Round-Trip Fidelity
Status: DONE
D-decisions touched: D5 (ruamel.yaml round-trip), D14 (byte_diff_drift gate)

## Why (from PRD)
D5 mandates ruamel.yaml in round-trip mode for all hieradata writes to preserve comments, anchors, and key order. D14's `byte_diff_drift` validation gate depends on being able to detect unintended changes by diffing the round-tripped YAML against the original byte-for-byte. This spike verifies ruamel.yaml produces byte-identical output (or only benign drift) on all alpin and dostoneu hieradata files.

## Assumptions
- Spike 4 (fixture capture) has completed and `tests/fixtures/alpin/` and `tests/fixtures/dostoneu/` exist.
- ruamel.yaml library is installed (`pip install ruamel.yaml`).
- Any drift observed is either byte-identical (perfect round-trip) or falls into a small set of benign patterns (e.g. trailing newline normalisation, comment spacing).
- If unexpected structural drift is found (key reordering, anchor mangling, comment loss), this spike fails and escalates to the architect.

## What to Investigate
For every `.yaml` file in `tests/fixtures/alpin/` and `tests/fixtures/dostoneu/`:
1. Load with `YAML(typ='rt').load(open(file))`
2. Dump with `YAML(typ='rt').dump(data, output_stream)`
3. Byte-diff the dumped output against the original file
4. Classify any diffs as:
   - **Perfect**: byte-identical (no diff)
   - **Benign**: known safe patterns (trailing newline, whitespace normalisation)
   - **Structural**: key reordering, anchor loss, comment loss → FAIL

Produce `bff/config/ruamel_tolerance.yaml`:
```yaml
# ruamel.yaml round-trip tolerance whitelist
# Generated by SPIKE-02 on <date>
# Tested against: tests/fixtures/{alpin,dostoneu}/**/*.yaml
benign_patterns:
  - pattern: "trailing_newline_added"
    description: "ruamel adds a trailing newline if missing"
    safe: true
  - pattern: "comment_spacing_normalised"
    description: "ruamel normalises comment indentation to match key depth"
    safe: true
  # ... (one entry per observed benign pattern)
unexpected_diffs: []  # If non-empty, this spike FAILS
verdict: "PASS"  # or "FAIL"
notes: |
  Any observations about ruamel behaviour, edge cases, or limitations.
```

## Pass Criteria
- All diffs are either byte-identical or match a benign pattern documented in `ruamel_tolerance.yaml`.
- `unexpected_diffs: []` (empty list).
- `verdict: "PASS"`.
- Tolerance file committed to `bff/config/ruamel_tolerance.yaml`.

## Fail Criteria
- Any file produces structural drift (key reorder, anchor mangle, comment loss).
- `unexpected_diffs` list is non-empty.
- `verdict: "FAIL"` → escalate to architect for resolution before D5/D14 implementation.

## Findings

Executed 2026-04-20 against synthetic representative fixtures in
`tests/fixtures/alpin/` (3 files) and `tests/fixtures/dostoneu/` (4 files).
SPIKE-04 (live fixture capture) had not yet run; fixtures were created to
cover the YAML features present in real hieradata: inline comments, leading
comments, block scalar literals (`|`), YAML anchors (`&`) and merge keys
(`<<: *`), quoted strings, nested mappings/sequences, and inline comments
with varying whitespace.

| File | Classification | Notes |
|---|---|---|
| alpin/common.yaml | Perfect | Byte-identical |
| alpin/roles/router.yaml | Perfect | Byte-identical |
| alpin/nodes/router01.alpin.example.com.yaml | Perfect | Byte-identical |
| dostoneu/common.yaml | Benign | `comment_spacing_normalised` — `# seconds` gained one leading space |
| dostoneu/env/devel.yaml | Perfect | Byte-identical |
| dostoneu/roles/switch.yaml | Perfect | Byte-identical |
| dostoneu/nodes/sw01.dostoneu.example.com.yaml | Perfect | Byte-identical |

Benign patterns observed: `comment_spacing_normalised` (one instance).
No anchor mangling, no key reordering, no comment loss, no block-scalar
coercion detected.

Key finding: `yaml.preserve_quotes = True` must be set on every `YAML(typ='rt')`
instance used in the write path to prevent unnecessary quote-stripping from
triggering the `byte_diff_drift` gate (D14).

**NOTE:** re-run `python scripts/spike_ruamel_roundtrip.py` once SPIKE-04
populates live fixtures. Verdict may be updated at that point.
