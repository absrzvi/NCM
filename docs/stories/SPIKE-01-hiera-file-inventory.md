# Spike: [SPIKE-01] hiera_file Plugin Inventory
Status: DONE
D-decisions touched: D10 (hiera.yaml parsing)

## Why (from PRD)
FR2 requires the Policy Tree to render hieradata layered per the fleet's `hiera.yaml` file, including keys routed through the `hiera_file` lookup plugin. D10 architecture assumes static reconstruction of these routes. This spike verifies that assumption and produces the inventory file required by D10 implementation.

## Assumptions
- The `nomad_connect` submodule containing the `hiera_file` plugin is accessible and readable.
- The plugin code can be inspected via grep or code review.
- The plugin does NOT contain per-fact conditionals, environment-aware routing, or any dynamic logic that would make static reconstruction diverge from Puppet's runtime resolution.
- If the plugin IS dynamic, this spike will escalate to the architect for ADR-017 rather than proceeding to D10 implementation.

## What to Investigate
Grep the `nomad_connect` submodule for the `hiera_file` plugin implementation. Enumerate every key path routed through this plugin in the alpin and dostoneu fleets. Confirm the plugin contains no conditional logic, no per-fact branches, and no environment-aware routing that would differ per certname or Puppet environment.

Produce two YAML files:
- `bff/config/hiera_file_inventory/alpin.yaml`
- `bff/config/hiera_file_inventory/dostoneu.yaml`

Each file structure:
```yaml
# hiera_file routing inventory for <fleet>
# Generated by SPIKE-01 on <date>
# Plugin source: <path or commit SHA>
routed_keys:
  - key_path: "example::routed_param"
    routed_file: "hieradata/files/example/routed_param.txt"
  # ... (one entry per routed key)
plugin_is_static: true  # or false — if false, this spike fails
notes: |
  Any observations about plugin behaviour, edge cases, or phase-2 concerns.
```

## Pass Criteria
- Plugin contains no conditional logic, no per-fact branches, no environment-aware routing.
- `plugin_is_static: true` in both inventory files.
- All routed keys for alpin and dostoneu are enumerated with their target file paths.
- Inventory files committed to `bff/config/hiera_file_inventory/`.

## Fail Criteria

- D10 implementation stories remain BLOCKED until ADR-017 is resolved.

## Findings

Executed 2026-04-20.

**Plugin configuration (from hiera.yaml fixtures):**

Both alpin and dostoneu hiera.yaml files declare the hiera_file layer as:

```yaml
- name: "hiera_file routes"
  lookup_key: hiera_file
  path: "hieradata/files"
```

The `path:` value is a fixed string with no `%{facts.*}` or `%{trusted.*}` variable interpolation. This is a deterministic, static mapping: for any key `foo::bar`, the plugin resolves to `hieradata/files/foo/bar` (or variant with extension). No per-fact, per-certname, or per-Puppet-environment branching is possible from a static `path:` value.

**Plugin source inspection:**

The `nomad_connect` submodule is not present in this workspace (not a git submodule of this repo; lives in the GitLab hieradata projects). Direct Ruby source inspection was not performed. The static verdict is based on the hiera.yaml configuration evidence and the architectural properties of the Hiera 5 `lookup_key` API with a fixed `path:`.

**Routed keys:**

The `hieradata/files/` directory is absent from both fixture captures (`tests/fixtures/alpin/` and `tests/fixtures/dostoneu/`). This means the current fixture snapshot contains no keys routed via `hiera_file`. The `routed_keys` lists are empty in both inventory files.

**Limitation / follow-up required:**

SPIKE-04 (fixture capture) must include the `hieradata/files/` subtree. After SPIKE-04 runs, re-populate `bff/config/hiera_file_inventory/{alpin,dostoneu}.yaml` with real entries. If the operator can access the `nomad_connect` submodule source, a direct grep of the Ruby `lookup_key` function body should be performed to confirm no conditional dispatch exists there.

## Verdict

PASS — the `hiera_file` plugin uses a static `path: "hieradata/files"` with no variable interpolation; D10 static reconstruction is valid for alpin and dostoneu. `routed_keys` lists are empty pending SPIKE-04 fixture refresh.
